# 导演定调（direction-tone）— 山音融合

> 摘自山音超级导演大师第一步「导演定调」，适配本仓库的 Wan2.7 工作流。

在编译 `storyboard.json` 之前，先做一次「导演定调」，把整集的视听
基调写进一份独立文件 `projects/<id>/<episode>/direction.json`。所有
shot prompt 必须从这份定调里长出来。

如果跳过这一步直接写 storyboard，**最常见的失败模式**是：调子
不统一、镜头随意、视觉没有母题、节奏波动失控。

---

## 一、什么是导演定调

导演定调 = **在拍摄之前，把整部作品的"调子"用决策清单固定下来**。
之后所有镜头设计都不再是"我现在想怎么拍"，而是"调子要求我怎么拍"。

定调有两个产物：
1. **七项视听决策**（光线 / 色彩 / 景别重心 / 运镜 / 对白处理 /
   节奏 / 转场）—— 锁定视听语言。
2. **意象体系**（视觉母题 + 强化元素）—— 锁定符号语言。

---

## 二、六维度判断（决定从哪些 genre 取武器）

读完 `script.md` 后，从下面六个维度判断本集**主导维度**和**调味维度**
（通常 1 主 + 1 调味）：

| 维度 | 关键词 | 例子 |
|------|--------|------|
| A 情绪基调 | 日常写实 / 情绪美学 / 诗意冥想 / 荒诞 / 悲剧 / 喜剧 / 黑色幽默 / 浪漫 | 武林外传 → 喜剧 |
| B 类型片 | 惊悚 / 心理 / 恐怖 / 犯罪 / 黑色电影 / 悬疑推理 / 科幻 / 奇幻 | 钱夫人挨揍 → 无 |
| C 动作与对抗 | 现代动作 / 武侠 / 战争 / 灾难 | 钱夫人挨揍 → 武侠 |
| D 题材关系 | 爱情 / 家庭 / 青春成长 / 传记 / 体育竞技 | 通常无 |
| E 叙事形式 | 伪纪录片 / 公路片 / 歌舞片 / 动画 / 实验先锋 | 通常无 |
| F 社会视角 | 社会现实主义 / 历史古装 / 政治讽刺 / 主旋律 | 古装 → 历史古装 |

**经验法则**：3 分钟短片基本只用 1 主导 + 1 调味。3 个以上维度
混在一起会让定调"什么都不是"。

## 三、七项视听决策（必填，写进 direction.json）

每一项都要给出**具体的、可执行的、有约束力的**答案。空话（"质感
要好"、"节奏要稳"）一律重写。

### 1. 光线设计

- 光源类型：日光散射 / 烛光 / 顶光 / 侧逆光 / 软光 / 硬光 …
- 高光区 vs 阴影区的比例：高调 / 中调 / 低调（明暗占比）
- 是否有标志性光位：例如"角色脸部永远是侧面 1/3 阴影"

例：「日光散射 + 暖黄烛光，中调（明暗 7:3），人物脸部右侧轻阴影」

### 2. 色彩计划

- 色板（3-5 个主色）：从 `lore.palette` 起步，可加调味色
- 主色 vs 强调色：例如"画面 80% 是暖黄褐，红色只在母题元素上出现"
- 饱和度：高 / 中 / 低
- 全片色彩弧光：是否有从冷到暖 / 从灰到艳的演进

例：「暖黄褐主调（80%）+ 锦绣红强调色（专留给母题），饱和度中高，
全片不做色彩弧光」

### 3. 景别重心

- 整体景别比例（全 / 中 / 近 / 特）
- 高潮段景别策略：例如"高潮全是大特写"或"高潮全是大全景"

例：「中景 50% + 近景 30% + 全景 15% + 特写 5%；高潮段切换到大特写」

### 4. 运镜偏好

- 静镜 vs 动镜的比例
- 主要运镜类型：推 / 拉 / 摇 / 跟 / 升降 / 手持
- 标志性运镜：是否有反复出现的同一运镜（作者签名）

例：「静镜 60% + 缓慢推近 30% + 跟拍 10%；手持完全禁用」

### 5. 对白处理

- 主用方式：正反打 / 单镜跟拍 / 过肩 / 镜面 / 动作中对话 / 旁白错位
- 哪些场景必须打破正反打惯性
- 沉默处理：留白 / 加环境音 / 配乐进

例：「30% 单镜双人 + 40% 过肩 + 30% 正反打；高潮场景禁用正反打」

### 6. 节奏策略

- 平均 shot 时长目标
- 三段节奏曲线（起 / 承 / 转 / 合 各段的密度）
- 呼吸点设计：在哪几个节点插一个 5-8s 的"喘气镜头"

例：「平均 12s/shot；起 15s/shot 缓 → 承 12s → 转 8s 紧 → 合 15s 顿；
转折点前插一个 5s 空镜」

### 7. 转场策略

- 主用：硬切 / 渐隐 / 叠化 / 时间跳切 / 空镜建场
- 哪些场景使用什么转场
- 是否有跨集呼应转场（剧集才需要）

例：「硬切为主；scene 切换处用一个 3s 空镜建场；不使用渐隐叠化」

## 四、意象体系（imagery_system）

继承自 `lore.imagery_system.motifs` 和 `highlight_elements`，但本集
要给每个 motif 制定**落地计划**：

```json
{
  "imagery_system": {
    "motifs": [
      {
        "name": "搓动的围裙",
        "meaning": "钱夫人的紧张与虚荣 —— 越搓越说明她心虚",
        "landings": ["S01-001", "S01-003", "S03-002"]
      },
      {
        "name": "翻飞的红盖头",
        "meaning": "重要时刻的视觉标记",
        "landings": ["S04-001"]
      }
    ],
    "highlight_elements": [
      {"name": "发饰特写", "shots": ["S01-002"]}
    ]
  }
}
```

**铁律**：
- 短片每个 motif 至少 2 次落地。少于 2 次 → 重新分配。
- motif 落地的 shot 必须把 motif **作为画面动作**写进 prompt，
  不是抽象提及。
- highlight_elements 至少要有 1 个特写 / 大特写镜头来强化。

## 五、direction.json 标准结构

```json
{
  "project_id": "my-own-swordsman-Mrs-Qian-get-punched",
  "episode_id": "episode-001",
  "tone_dimensions": {
    "primary": "A:喜剧",
    "secondary": "C:武侠",
    "tertiary": "F:历史古装"
  },
  "director_reference": "宁浩式群像喜剧节奏 + 张艺谋色彩饱和度",
  "audiovisual_decisions": {
    "lighting": "日光散射 + 暖黄烛光，中调...",
    "color": "暖黄褐主调 80% + 锦绣红强调色...",
    "shot_size_distribution": "中 50 / 近 30 / 全 15 / 特 5",
    "camera_movement": "静镜 60% + 缓推 30% + 跟 10%",
    "dialog_handling": "30% 单镜 + 40% 过肩 + 30% 正反打",
    "pacing_strategy": "平均 12s/shot, 起缓承中转紧合顿",
    "transition_strategy": "硬切为主, scene 间 3s 空镜"
  },
  "imagery_system": {
    "motifs": [
      {
        "name": "搓动的围裙",
        "meaning": "...",
        "landings": ["S01-001", "S01-003", "S03-002"]
      }
    ],
    "highlight_elements": [
      {"name": "发饰特写", "shots": ["S01-002"]}
    ]
  },
  "double_check_targets": {
    "tone_consistency": "整集 prompt 必须出现暖黄烛光",
    "pacing_match": "起承段不超过 12s/shot, 转段必须有 ≤8s 短镜",
    "highlight_design_ratio": ">=20% shots 含非常规元素"
  }
}
```

## 六、自检清单（每写完 direction.json 走一遍）

- [ ] 七项视听决策每一项都具体可执行（无空话）？
- [ ] 主导 + 调味维度判断清晰，没有"什么都沾点"？
- [ ] director_reference 引用了 1-2 位真实导演的方法论？
- [ ] 每个 motif 都有 ≥2 个 landings 计划（短片）？
- [ ] highlight_elements 至少 1 个特写计划？
- [ ] double_check_targets 给了可量化的目标？

---

## 七、与 storyboard.json 的关系

direction.json 是 storyboard.json 的"上位约束"：

```
lore.md  →  direction.json  →  storyboard.json
(项目级)    (本集导演定调)     (本集 shot 序列)
```

写 storyboard 时：
1. 每个 shot 的 prompt 必须服从 direction 的七项决策。
2. `imagery_system.motifs[].landings` 里列出的 shot id 必须真的把
   motif 写进 prompt。
3. `narrative_purpose` 写完后，回头看是否服从 `pacing_strategy` 和
   `dialog_handling`。
